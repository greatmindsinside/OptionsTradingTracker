## üìù Documentation Guidelines

When updating documentation:

1. **Keep it current** - Update dates and version references
2. **Be concise** - Clear and direct language
3. **Use examples** - Code snippets and screenshots when helpful
4. **Cross-reference** - Link to related docs
5. **Archive old content** - Move outdated docs to `archive/`

### Which Doc to Update?

- **User-facing features** ‚Üí `FEATURES.md`
